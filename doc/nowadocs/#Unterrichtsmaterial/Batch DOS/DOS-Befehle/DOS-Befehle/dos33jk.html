IBMAVSP

IBM AntiVirus

Eigenständige Version von IBMAVSP verwenden.

Syntax: IBMAVSP [optionen]

Werden keine Optionen angegeben, arbeitet die eigenständige Version von

IBM AntiVirus interaktiv. Die meisten Benutzer bevorzugen diese Arbeitsweise.

Der Einführung zu IBM AntiVirus sind nähere Informationen zur

eigenständigen Version zu entnehmen.

Optionen werden nachfolgend beschrieben. HINWEIS: Lange

Optionen können mit den ersten drei Buchstaben abgekürzt werden.

? : Diese Hilfeanzeigen aufrufen.

* : Alle lokalen Festplattenlaufwerke auf Viren überprüfen.

Bei Verwendung dieser Option wird der Benutzer von

der eigenständigen Version von IBM AntiVirus nicht

aufgefordert, die zu prüfenden Laufwerke anzugeben.

*N : Alle Netzlaufwerke überprüfen. Bei Verwendung dieser Option

wird der Benutzer von der eigenständigen Version von IBM

AntiVirus nicht aufgefordert, zu überprüfende Laufwerke

anzugeben.

C: : Laufwerk C überprüfen. Auch andere Laufwerkbuchstaben können

angegeben werden. Werden in der Befehlszeile ein oder

mehrere Laufwerkbuchstaben angegeben, wird der Benutzer von

der eigenständigen Version von IBM AntiVirus nicht

aufgefordert, die zu überprüfenden Laufwerke anzugeben.

-LOGlogfilename : Der eigenständigen Version von IBM AntiVirus angeben, wohin

die Protokolldatei gestellt werden soll. Bei Verwendung

dieser Option wird der Benutzer von der eigenständigen

Version von IBM AntiVirus nicht aufgefordert, anzugeben,

wohin die Protokolldatei gestellt werden soll.

"logfilename" ist wahlfrei und kann ein vollständiger Pfad

sein. Wird "logfilename" nicht angegeben, verwendet die

eigenständige Version von IBM AntiVirus den

Standarddateinamen "IBMAVSP.LOG" im aktuellen Verzeichnis.

Um die Kompatibilität mit der vorherigen Version der

eigenständigen Version von IBM AntiVirus zu gewährleisten,

kann "-SPL" anstelle von "-LOG" angegeben werden.

-ALLFILES : Alle Dateien auf angegebenen Laufwerken auf Viren

überprüfen. Bei Verwendung dieser Option wird der Benutzer

von der eigenständigen Version von IBM AntiVirus nicht

aufgefordert, anzugeben, ob alle Dateien oder nur Programme

überprüft werden sollen.

-PROGRAMS : Nur Programme auf angegebenen Laufwerken auf Viren

überprüfen. Bei Verwendung dieser Option wird der

Benutzer von der eigenständigen Version von IBM AntiVirus

nicht aufgefordert, anzugeben, ob alle Dateien oder

nur Programme überprüft werden sollen.

-VLOG : Die Namen aller überprüften Objekte in die Protokolldatei

aufnehmen, unabhängig davon, ob sie infiziert sind.

-NB : Bei Feststellung eines Virus kein Tonsignal ausgeben.

-NLOG : Keine Protokolldatei erstellen.

-COPENERR : Die Virusprüfung ohne Bestätigung durch den Benutzer

fortsetzen, auch wenn eine Datei nicht geöffnet werden

kann.

-CERR : Die Virusprüfung ohne Bestätigung durch den Benutzer

fortsetzen, wenn ein behebbarer Fehler auftritt.

-MEM : Hauptspeicher auf Viren überprüfen. Die eigenständige

Version von IBM AntiVirus fordert nicht zur Eingabe der zu

überprüfenden Laufwerke und Dateitypen auf. Der

Hauptspeicher wird immer überprüft. Diese Option kann

angegeben werden, wenn die Überprüfung des Hauptspeichers

ausreicht.

-ONEFLOP : Überprüfung einer weiteren Diskette nicht anbieten. Die

eigenständige Version von IBM AntiVirus fordert den Benutzer

normalerweise auf, anzugeben, ob eine weitere Diskette

überprüft werden soll.

Die folgenden sechs Optionen (-NREP, -NWIPE, -NFSCAN, -YREP, -YWIPE,

-YFSCAN) beeinflussen, wie die eigenständige Version von IBM AntiVirus

bei Feststellung infizierter Objekte reagiert. Normalerweise bietet IBM

AntiVirus an, infizierte Objekte zu korrigieren, wenn es die

Korrekturmethode kennt. Außerdem bietet das Programm an, infizierte

Dateien zu überschreiben und infizierte Urladeprogrammsätze zu ersetzen,

die es noch nicht korrigiert hat. Werden bei der Prüfung infizierte

Objekte gefunden, bietet es an, alle Dateien auf allen lokalen

Laufwerken zu überprüfen.

-NREP : Infizierte Dateien/Urladeprogrammsätze nicht korrigieren.

-NWIPE : Infizierte Dateien nicht löschen, infizierte

Urladeprogrammsätze nicht ersetzen.

-NFSCAN : Dem Benutzer bei Feststellung von Viren nicht anbieten,

eine umfassende Überprüfung durchzuführen.

-YREP : Infizierte Dateien/Urladeprogrammsätze korrigieren.

-YWIPE : Infizierte Dateien löschen, infizierte Urladeprogrammsätze

ersetzen. Dateien/Urladeprogrammsätze werden nur dann

gelöscht/ersetzt, wenn sie nicht bereits korrigiert wurden.

-YFSCAN : Bei Feststellung von Viren wird eine umfassende Prüfung

durchgeführt, ohne daß die Bestätigung des Benutzers

angefordert wird.

Beispiele:

Zur interaktiven Ausführung der eigenständigen Version von IBM AntiVirus

folgenden Befehl eingeben und Benutzerführungsanzeigen beachten.

Die meisten Benutzer ziehen diese Arbeitsweise vor.

IBMAVSP

Soll IBMAVSP alle Programme auf allen lokalen Festplattenlaufwerken

überprüfen und als Protokolldatei "C:\IBMAVSP.LOG" verwenden, folgenden

Befehl eingeben. Diese Vorgehensweise empfiehlt sich bei

.BAT-Dateien und immer dann, wenn keine Interaktion erforderlich ist.

ibmavsp * -programs -logc:\ibmavsp.log
